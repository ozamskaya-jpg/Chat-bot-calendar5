# 🤖 Чат-бот для записи клиентов

Telegram-бот для онлайн-записи клиентов с уведомлениями и HTTP API для интеграции с внешним календарём.

## Возможности

**Для клиентов (через Telegram):**
- 📅 Записаться на приём (выбор даты и времени из доступных слотов)
- 🔄 Перенести запись на другую дату/время
- 📋 Просмотреть свои активные записи
- ❌ Отменить запись
- 🔔 Автоматические напоминания за 24 часа и за 1 час до приёма

**Для владельца (через Telegram):**
- ⚙️ `/settings` — интерактивное меню настройки расписания
- 📥 Импорт записей через JSON-файл (отправить файл `.json` боту)

**HTTP API (для внешних приложений/календаря):**
- Получение событий, свободных слотов, данных календаря
- Создание, удаление событий и бронирований
- Управление настройками и отпусками

## Переменные окружения (Railway → Variables)

| Переменная | Описание |
|---|---|
| `BOT_TOKEN` | Токен бота от @BotFather |
| `OWNER_ID` | Ваш Telegram ID (узнать у @userinfobot) |
| `DATABASE_URL` | Строка подключения PostgreSQL |
| `API_TOKEN` | Токен для HTTP API (по умолчанию: `changemesecrettoken`) |
| `PORT` | Порт HTTP-сервера (по умолчанию: `8080`) |

## Настройки расписания (через `/settings`)

| Параметр | Описание | По умолчанию |
|---|---|---|
| Рабочие дни | Пн–Пт (можно выбрать любые) | 1–5 |
| Рабочее время | Начало и конец рабочего дня | 09:00–18:00 |
| Обед | Перерыв (или «нет») | 13:00–14:00 |
| Длительность слота | 15 / 20 / 30 / 45 / 60 / 90 / 120 мин | 60 мин |
| Отпуск/выходные | Период дат, когда запись недоступна | — |

## HTTP API

Все запросы требуют токен: `?token=API_TOKEN` (GET/DELETE) или поле `"token"` в теле (POST).

| Метод | Путь | Описание |
|---|---|---|
| GET | `/health` | Проверка работоспособности |
| GET | `/events` | Список событий (`?month=YYYY-MM` — опционально) |
| GET | `/slots?date=YYYY-MM-DD` | Свободные слоты на дату |
| GET | `/calendar-data` | Все данные: настройки + отпуска + события |
| POST | `/book` | Забронировать слот (проверяет доступность) |
| POST | `/event` | Создать событие любого типа без проверки слотов |
| DELETE | `/event?id=ID` | Удалить событие по ID |
| POST | `/calendar-settings` | Обновить настройки и отпуска |
| POST | `/vacation` | Добавить период отпуска |
| DELETE | `/vacation?id=ID` | Удалить период отпуска |

### Типы событий
- `c` — клиентская запись (booking)
- `b` — заблокированное время
- `e` — другое событие

### Пример POST /book
```json
{
  "token": "your_api_token",
  "date": "2026-06-20",
  "time": "10:00",
  "name": "Иван Иванов",
  "phone": "+7 999 123-45-67",
  "note": "Комментарий"
}
```

### Пример POST /event
```json
{
  "token": "your_api_token",
  "date": "2026-06-20",
  "starttime": "11:00",
  "endtime": "12:00",
  "name": "Название",
  "type": "b"
}
```

## Импорт данных

Владелец может отправить боту `.json`-файл с массивом записей для импорта:

```json
{
  "evs": [
    {
      "t": "c",
      "n": "Имя клиента",
      "d": "2026-06-20",
      "st": "10:00",
      "en": "11:00",
      "ph": "+7 999 000-00-00",
      "no": "Примечание"
    }
  ]
}
```

Дублирующиеся записи (по `id`) пропускаются автоматически.

## Архитектура

- **Telegram-бот** — `python-telegram-bot` с ConversationHandler
- **HTTP API** — `aiohttp`, работает параллельно с ботом
- **База данных** — PostgreSQL (таблицы: `settings`, `events`, `vacations`)
- **Напоминания** — фоновая задача, проверяет каждые 5 минут
- **Защита от дублирования** — PostgreSQL advisory lock предотвращает одновременную работу двух экземпляров бота
